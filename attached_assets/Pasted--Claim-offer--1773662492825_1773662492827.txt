Writing
You are a senior backend engineer and Discord bot architect.

Your task is to build a professional Discord bot for managing a manhwa scanlation team called "Team X".

This bot must be production-ready, modular, and scalable.

The bot will manage translation workflow, raw chapter tracking, project channels, team member profiles, and claim systems.

The bot must use Node.js with discord.js v14 and a PostgreSQL database.

The code must be clean, modular, and well-structured.

GENERAL REQUIREMENTS

• Use Slash Commands only
• No prefix commands
• No automatic thread creation
• Each project must have its own Discord channel
• Only admins can create projects or open chapters
• Members interact mainly using buttons

TEAM ROLES

The team has three main roles:

TL = Translator
ED = Editor
TR = Trainee

Workflow of chapters:

RAW
↓
TL (translation)
↓
ED (cleaning + typesetting)
↓
Release

Trainees can review translations for training but they are not part of the main workflow.

RAW CHAPTER DETECTION SYSTEM

The bot must track new raw chapters from sites such as:

NewToki
Asura Scans
Vortex Scans
Nirvana Scans

Use web scraping with Puppeteer or Cheerio.

Every few minutes the bot checks for new chapters.

When a new chapter is detected:

Post a notification in the project channel

Include the RAW link

Optionally download images if the chapter is free

RAW DOWNLOADER

If the chapter is free:

The bot should:

Extract image URLs

Download images

Compress them

Upload them to Google Drive

Send the Drive link in the channel

If the chapter is paid:
Only send the raw link.
